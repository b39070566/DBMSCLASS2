# CPBL 戰績系統 (CPBL Standings System)

設計一個中華職棒聯盟的戰績系統（假設性的迷你世界），該系統可以記載球隊資訊和比賽記錄 。


## 開發工具與技術棧

本系統的後端開發採用 Python Django ，搭配 HTML 與 CSS 建構前端視圖 。資料庫管理系統使用 PostgreSQL 並以 pgAdmin 進行操作 。開發環境主要在 Visual Studio Code 中進行 ，版本控制依賴 GitHub ，最後將專案部署至 Render 平台 。

## 系統功能

系統主要分為前端使用者介面與後台管理端 。

在前端部分，使用者可以進行帳號註冊與登入 。登入後可以查看戰績查詢頁面，檢視各球隊的出賽數、勝敗場、勝率與勝差 。同時提供賽局查詢功能，讓使用者能檢視過往比分與詳細比賽資訊 。使用者也能透過隊伍查詢頁面瀏覽隊伍基本資料、隊員與教練資訊 ，或是透過專屬的人員查詢頁面（點將錄）來檢閱球員與教練的詳細資料 。

在後台管理端，管理者擁有完整的增刪改查權限 。管理者可以進行隊伍管理，包含編輯隊伍名稱、公司聯絡資訊與主場設定 。球員管理則允許編輯球員的背號、姓名、生日、守備位置、身高、體重與學歷 。此外，也包含教練管理、賽局紀錄維護，以及球場名稱與地址的場地管理功能 。

## 資料庫設計

本專案的關聯式資料庫設計包含五個主要實體。

球隊（Team）實體記錄隊名、總教練、所屬公司名稱、公司電話與公司地址 。教練（Coach）實體記錄教練號碼、姓名與生日 。球員（Player）實體負責記錄球衣號碼、姓名、生日、位置、身高、體重與學歷，每一位球員都必須隸屬於一個球隊 。比賽（Game）實體負責記載勝隊、敗隊、比賽日期、比數與比賽球場 。最後，球場（Field）實體則包含球場 ID、名稱與地址 。

## 系統部署與測試帳號

本專案目前部署於 Render 平台，網址為 [https://dbmsclass2.onrender.com](https://dbmsclass2.onrender.com) 。由於伺服器設定，初次連線的跳轉喚醒時間約需等待一分鐘左右 。

系統提供兩組測試帳號供檢閱。若要測試前端使用者端的功能，如賽局查詢、戰績與點將錄，可以使用帳號 M144020015 搭配密碼 0125 登入 。若要進入後台管理端檢視各項資料的增減維護功能，請使用管理員帳號 admin 搭配密碼 12345 登入 。
